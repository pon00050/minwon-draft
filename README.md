# minwon-draft

사실관계만 입력하면 제출할 수 있는 **국민신문고 민원·안전신문고 신고서 초안**을 만들어 주는 [Claude Code](https://claude.com/claude-code) 스킬입니다. 인용한 법령은 law.go.kr에서 실시간으로 확인하고, 제출 전에 초안을 스스로 점검합니다.

## 무엇을 하나요

쟁점, 대상 기관, 관련 법령 등 현재 알고 있는 사실관계를 입력하면 다음 내용을 만듭니다.

- 제출용 **제목**과 **내용** (국민신문고 입력칸은 일반 텍스트라 서식 없이 작성합니다)
- 권장 민원종류·기관·신청 채널
- 인용한 법령의 확인 결과표 (시행일자 포함)
- 기관 답변에 따라 이어서 무엇을 할지 정리한 대응표

## 주요 기능

- **결론을 먼저 제시하고 확인을 요청합니다.** 열린 질문만 던지는 대신, 잠정적인 해석을 먼저 제시하고 "이 이해가 맞는지" 확인을 요청합니다. 기관이 어떤 부분을 인정·부정·유보하는지 명확히 확인할 수 있고, 부정적인 답변도 후속 대응의 근거로 활용할 수 있습니다.
- **인용한 법령을 실시간으로 확인합니다.** 인용하는 모든 조문을 [korean-law-mcp](https://github.com/chrisryugj/korean-law-mcp) 커넥터로 law.go.kr 현행 데이터와 대조합니다(시행일자 포함). 기억이나 웹 검색에 의존하지 않습니다.
- **제출 전 반론 가능성을 점검합니다.** 담당자 입장에서 반려하거나 회피할 여지가 있는지 초안을 다시 점검합니다.
- **안전신문고의 입력 조건을 미리 반영합니다.** 내용 글자 수(약 1,600자, 관측값), 차량번호 등 별도 입력란에서 안내 없이 내용이 잘릴 수 있는 점, 자동 과태료용 사진 촬영 간격을 초안 단계에서 고려합니다.

동봉된 예시(전자금융거래법 해당 여부, 장애인 차별 진정, 교통 안전신고)는 방식을 보여주기 위한 것입니다. 특정 분야에 한정하지 않고 다양한 민원에 동일한 작성 원칙을 적용할 수 있도록 설계했으며, 실제 사안에서는 법령과 용어를 매번 새로 확인합니다.

## 동작 순서

1. 사실관계 정리·분류
2. 인용 법령 실시간 확인
3. 초안 작성 (제목 + 내용)
4. 반론·회피 가능성 점검
5. 최종 초안과 답변 유형별 대응표 제공

접수 후 신청번호를 받으면 제출 내용과 진행 경과를 보관용 기록(.md)으로 정리할 수 있습니다.

## 이 스킬이 지향하는 것

이 스킬은 민원인이 자신의 사실관계와 질문을 명확하게 정리하고, 행정기관도 무엇을 확인하거나 설명해야 하는지 분명하게 드러나도록 돕기 위해 만들어졌습니다. 결과적으로, 사실과 근거를 중심으로 정리된 민원은 불필요하게 길거나 감정적인 민원보다 담당자의 검토에도 도움이 됩니다.

행정기관은 국민에게 이해하기 쉬운 설명을 제공할 책임이 있고, 민원인은 정확한 사실관계와 확인받고 싶은 내용을 분명하게 제시할 책임이 있습니다. 어느 한쪽을 압박하거나 부담을 늘리기보다, 양측이 같은 쟁점을 바라보고 기록으로 남길 수 있도록 하는 데 목적이 있습니다.

> 좋은 민원은 원하는 답을 얻는 민원이 아니라, **어떤 답이 오더라도 다음 판단과 행동의 근거가 남는 민원**이라고 생각합니다. "결론을 먼저 제시하고 확인을 요청하는" 방식은 여기서 나왔습니다.

다만 이 스킬이 작성한 초안이 항상 옳은 것은 아닙니다. 사실관계가 잘못 입력될 수도, 법령 해석에 여러 견해가 있을 수도 있고, 처리기한이나 이의신청·행정심판 기간은 민원 유형에 따라 다릅니다. 최종적인 법적 판단은 관계 기관이나 법원의 몫입니다. 제출 전에는 사실관계와 인용한 조문이 정확한지, 그리고 자신이 실제로 확인받고 싶은 내용이 맞는지 한 번 더 살펴보시길 권합니다. 아직 초기(v0.1) 단계이므로 생성된 내용은 초안으로 보시기 바랍니다.

## 설치

Claude Code가 필요합니다.

### 1. 법제처 API 키 발급 (무료)

인용 확인에 쓰는 법제처 Open API 인증키(`LAW_OC`)를 먼저 받으세요.

1. [법제처 Open API 신청](https://open.law.go.kr/LSO/openApi/guideList.do)에 접속합니다.
2. 회원가입·로그인 후 "Open API 사용 신청"을 누릅니다.
3. 발급된 인증키를 아래 설치 과정에서 입력합니다.

키가 없어도 초안은 작성됩니다. 대신 각 법령 인용에 `⚠️ 미검증` 표시가 붙으니, 제출 전 [law.go.kr](https://www.law.go.kr)에서 직접 확인하세요. 나중에 키를 넣으면 자동 확인이 켜집니다.

### 2. 플러그인 설치

```
/plugin marketplace add pon00050/minwon-draft
/plugin install minwon-draft@minwon-draft-marketplace
```

설치 중 법제처 API 키를 입력하는 창이 나옵니다. 민감정보로 안전하게 저장되며 law.go.kr 호출에만 사용됩니다.

## 사용법

Claude Code에서 자연어로 요청하거나 스킬을 직접 실행할 수 있습니다.

```
/minwon-draft:minwon-draft <사실관계>
```

예를 들어 다음과 같이 요청할 수 있습니다.

- "집 앞 소화전에 계속 주차하는 차가 있어요. 불법주정차 신고 내용 좀 써 주세요."
- "이 내용으로 국민신문고 민원 초안 만들어 주세요: …"
- "○○부 고시 해석을 묻는 질의를 쓰고 싶어요."

지원 범위는 국민신문고 일반민원·고충민원과 안전신문고 안전신고입니다. 문서24 공문, 정보공개청구, 행정심판은 다음 단계로 안내만 하고 문서 작성은 아직 지원하지 않습니다.

## 법령 인용은 어떻게 확인하나요

법령 인용 확인은 [korean-law-mcp](https://github.com/chrisryugj/korean-law-mcp)(MIT)가 담당합니다. 법제처 국가법령정보 Open API를 감싼 MCP 서버로, 이 플러그인에 번들로 포함되어 있어 플러그인을 설치하면 서버 설정도 함께 등록됩니다. 법제처 API 키 외에는 별도의 MCP 설정이 필요하지 않습니다.

스킬이 사용하는 도구는 다음과 같습니다.

- `search_law` — 법령 존재 여부와 시행일자·MST 조회
- `get_law_text` — 조문 전문을 그대로 인용
- `legal_analysis`(`mode: verify_citations`) — 조문 존재 여부와 잘못된 조문 제목 검사
- `search_admin_rule` / `get_admin_rule` — 행정규칙(고시·훈령·예규)
- `search_precedents` / `get_precedent_text` — 판례

이미 korean-law-mcp를 커넥터로 쓰고 있다면 그대로 동작합니다.

## 문제 해결

- **설치 직후 "korean-law 연결 실패"가 뜰 때** — 처음 실행할 때 npx가 korean-law-mcp를 내려받는 동안 잠시 실패로 보일 수 있습니다. 다운로드가 끝나면 연결됩니다(이후로는 바로 연결). 계속 실패하면 Node.js 18 이상 설치 여부(`node --version`)와 npm 네트워크 접근을 확인하세요.
- **인용에 `⚠️ 미검증`이 붙을 때** — `LAW_OC` 키가 설정되지 않은 경우입니다. 플러그인 구성에서 무료 키를 넣으세요.

## 개인정보

- `LAW_OC` 키는 Claude Code가 민감정보로 로컬에 저장하며 저장소에 올라가지 않습니다.
- 초안과 기록은 사용자 컴퓨터에만 남습니다. `.gitignore`가 `국민신문고/` 기록과 `.env`를 기본으로 제외합니다.
